# chn-igame-lab

## 项目简介

`chn-igame-lab` 是一个用于归档和发布独立 HTML 页面的仓库。这里存放的是各类实验性、演示性或功能性的单页 HTML 文件，不针对特定域名或网站。你可以将本仓库视为一个静态 HTML 页面的集合与发布源。

## 目录说明

```
.
├── README.md           # 本文件
├── pages/              # 所有独立 HTML 页面存放目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/             # 公共资源文件（如样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：每个 HTML 文件是一个独立的页面，可直接通过浏览器打开或部署后访问。
- `assets/`：存放页面共用的静态资源，便于复用与维护。

## 页面归档说明

- 每个 HTML 页面力求自包含，尽量减少对外部资源的依赖。
- 归档的页面可能包含实验性代码、原型演示、工具页面或学习笔记。
- 页面内容不代表任何官方立场，仅作为个人或协作项目存档。

## 维护说明

- 欢迎通过 Pull Request 提交新的独立 HTML 页面，或将现有页面更新至`pages/`目录。
- 请确保提交的页面不包含敏感信息、恶意代码或违反法律法规的内容。
- 如需引用外部资源，建议优先使用相对路径或本仓库`assets/`目录下的文件。
- 本仓库不承诺长期在线或可用性，请自行备份重要内容。

## 使用方式

你可以通过以下方式使用本仓库中的页面：

1. **直接浏览**：克隆仓库到本地，用浏览器打开`pages/`目录下的 HTML 文件。
2. **部署到静态托管服务**：将整个仓库或`pages/`目录部署到任何支持静态文件的托管平台（如 GitHub Pages、Netlify、Vercel 等）。
3. **嵌入或引用**：在遵守页面自身许可的前提下，可将页面内容嵌入到其他项目中使用。

## 贡献指南

- 提交新页面时，请在`pages/`下新建一个以页面功能或主题命名的 `.html` 文件。
- 建议在页面头部添加简要注释，说明页面用途、作者及依赖资源。
- 若页面使用外部库或 API，请在注释中注明来源与版本。
- 所有贡献均视为同意以 MIT 许可证（或页面内指定的许可证）发布。
